890. Find and Replace Pattern
❓ What is the question asking?

You are given:

An array of words words
A pattern string pattern

Your task is to return all words that follow the same pattern.

A word matches the pattern if there is a one-to-one mapping between the characters of the pattern and the characters of the word.

Each pattern character should map to exactly one character, and each character should map to exactly one pattern character.

🪨 Brute Force Approach
💡 Idea

For every word:

Compare every character with every other character.
Check whether equal pattern characters map to equal word characters.
Check whether different pattern characters map to different word characters.
⏱️ Time Complexity
O(n × m²)

where

n = number of words
m = length of each word
📦 Space Complexity
O(1)
⚡ Optimal Approach
💡 Idea

For every word:

If the word length is different from the pattern length, return false.
Use a HashMap to map:
Pattern character → Word character
Use a HashSet to ensure:
One word character is mapped only once.
Traverse both strings together.
If every mapping is valid, add the word to the answer list.
💻 Java Code
import java.util.*;

class Solution {

    public List<String> findAndReplacePattern(String[] words, String pattern) {

        List<String> ans = new ArrayList<>();

        for (String word : words) {
            if (same(word, pattern)) {
                ans.add(word);
            }
        }

        return ans;
    }

    public boolean same(String word, String pattern) {

        if (word.length() != pattern.length()) {
            return false;
        }

        HashMap<Character, Character> map = new HashMap<>();
        HashSet<Character> set = new HashSet<>();

        for (int i = 0; i < word.length(); i++) {

            char p = pattern.charAt(i);
            char w = word.charAt(i);

            if (!map.containsKey(p)) {

                if (set.contains(w)) {
                    return false;
                }

                map.put(p, w);
                set.add(w);

            } else {

                if (map.get(p) != w) {
                    return false;
                }
            }
        }

        return true;
    }
}

⭐ Interview Tip

This problem is an extension of 290. Word Pattern.

Whenever you see:

"Pattern matching"
"One-to-one mapping"
"Bijective relationship"

Think:

HashMap + HashSet
HashMap → Stores pattern character → word character.
HashSet → Prevents two pattern characters from mapping to the same word character.

This HashMap + HashSet technique is the standard interview solution with O(n × m) time complexity and is considered optimal. 💖📚

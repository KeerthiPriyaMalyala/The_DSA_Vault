7. Reverse Integer
❓ What is the question asking?

You are given a 32-bit signed integer x.

Reverse its digits and return the reversed integer.

If reversing the integer causes it to go outside the 32-bit signed integer range [-2³¹, 2³¹ - 1], return 0.

Example 1

Input:

x = 123

Output:

321
Example 2

Input:

x = -123

Output:

-321
Example 3

Input:

x = 120

Output:

21
🪨 Brute Force Approach
💡 Idea
Convert the integer into a String.
Reverse the string.
Convert it back to an integer.
Handle the negative sign separately.

❌ This approach is not preferred because:

Extra space is used.
The problem expects you to solve it without converting to a string.
⚡️ Optimal Approach
💡 Idea
Extract the last digit using % 10.
Add it to the reversed number.
Remove the last digit using / 10.
Before updating the reversed number, check for integer overflow.
If overflow occurs, return 0.
💻 Java Code
class Main {

    public static int reverse(int x) {

        int rev = 0;

        while (x != 0) {

            int digit = x % 10;

            if (rev > Integer.MAX_VALUE / 10 ||
                (rev == Integer.MAX_VALUE / 10 && digit > 7))
                return 0;

            if (rev < Integer.MIN_VALUE / 10 ||
                (rev == Integer.MIN_VALUE / 10 && digit < -8))
                return 0;

            rev = rev * 10 + digit;
            x /= 10;
        }

        return rev;
    }

    public static void main(String[] args) {

        int x = 123;

        System.out.println(reverse(x));
    }
}
▶️ Output
321
⏱️ Time Complexity
O(log₁₀ n)
We process each digit exactly once.
📦 Space Complexity
O(1)
Only a few variables are used.

📝 Interview Notes 💖
✅ Use % 10 to get the last digit.
✅ Use / 10 to remove the last digit.

✅ Build the reversed number using:

rev = rev * 10 + digit;
✅ Always check for overflow before updating rev.
✅ If overflow occurs, return 0.
❌ Do not use string conversion in the optimal solution.
✅ Time Complexity: O(log₁₀ n)
✅ Space Complexity: O(1)
⚠️ Overflow Check
if (rev > Integer.MAX_VALUE / 10 ||
    (rev == Integer.MAX_VALUE / 10 && digit > 7))
    return 0;

if (rev < Integer.MIN_VALUE / 10 ||
    (rev == Integer.MIN_VALUE / 10 && digit < -8))
    return 0;

Why 7 and -8?

Integer.MAX_VALUE = 2147483647 → last digit is 7
Integer.MIN_VALUE = -2147483648 → last digit is -8
🎯 Interview One-Liner

Extract each digit using % 10, build the reversed number using rev = rev * 10 + digit, and check for integer overflow before every update. 💖🚀

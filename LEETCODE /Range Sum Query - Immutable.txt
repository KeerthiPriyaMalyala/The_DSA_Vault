303. Range Sum Query - Immutable
❓ What is the question asking?

You are given an integer array nums.

You have to answer multiple queries.

Each query contains:

left
right

Return the sum of all elements from index left to right (inclusive).

Note: The array never changes (Immutable). Only queries are performed.

🪨 Brute Force Approach
💡 Idea

For every query:

Traverse from left to right.
Add all elements.
Return the sum.
💻 Java Code
class NumArray {

    int[] nums;

    public NumArray(int[] nums) {
        this.nums = nums;
    }

    public int sumRange(int left, int right) {

        int sum = 0;

        for (int i = left; i <= right; i++) {
            sum += nums[i];
        }

        return sum;
    }
}
▶️ Output
Input:
nums = [-2,0,3,-5,2,-1]

sumRange(0,2)

Output:
1
⏱️ Time Complexity
Constructor : O(1)

Each Query : O(n)
📦 Space Complexity
O(1)
⚡ Optimal Approach (Prefix Sum)
💡 Idea

Instead of calculating the sum every time,

Create a Prefix Sum Array.

nums

1 2 3 4 5

↓

prefix

1 3 6 10 15

Now,

If

left == 0

Answer is

prefix[right]

Otherwise,

prefix[right] - prefix[left - 1]

This makes every query O(1).

💻 Java Code
class NumArray {

    int[] prefix;

    public NumArray(int[] nums) {

        prefix = new int[nums.length];

        prefix[0] = nums[0];

        for (int i = 1; i < nums.length; i++) {
            prefix[i] = prefix[i - 1] + nums[i];
        }
    }

    public int sumRange(int left, int right) {

        if (left == 0)
            return prefix[right];

        return prefix[right] - prefix[left - 1];
    }
}
▶️ Output
Input:
nums = [-2,0,3,-5,2,-1]

sumRange(0,2)

Output:
1
Input:
sumRange(2,5)

Output:
-1
Input:
sumRange(0,5)

Output:
-3
⏱️ Time Complexity
Constructor : O(n)

Each Query : O(1)
📦 Space Complexity
O(n)

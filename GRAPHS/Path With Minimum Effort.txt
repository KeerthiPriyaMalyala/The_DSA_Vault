🚀 Path With Minimum Effort
❓ What is the question asking?

We are given:

A grid of heights.
Start from:

Top Left → (0,0)

Reach:

Bottom Right → (n-1,m-1)

We need to find:

The minimum effort required to reach the destination.

🌟 What is Effort?

Effort is NOT the total sum of differences.

Instead,

For every path,

calculate:

Maximum Absolute Height Difference
between consecutive cells.

Among all possible paths,

return the minimum such maximum.

🌟 Main Idea

Moving to a neighbour has a cost:

|currentHeight - neighbourHeight|

But,

Path Cost =

Maximum edge cost

not

Sum of edge costs.

So,

Normal BFS ❌

Normal DFS ❌

Dijkstra ✅

🌟 Why Dijkstra?

Normally,

Dijkstra stores

Shortest Distance

Here,

we store

Minimum Effort

Whenever we move,

New Effort =

max(Current Effort,
Current Height Difference)

Always choose the path with the smallest effort first.

⚡ Optimal Approach (Dijkstra)
💡 Idea
Create an effort matrix.
Initialize every cell with
∞
Set
effort[0][0] = 0
Use a Priority Queue.

Store

(row, col, effort)
Pop the cell having the minimum effort.
Visit all four neighbours.
Calculate
heightDifference =
abs(currentHeight - neighbourHeight)
New Effort
max(currentEffort,
heightDifference)
If new effort is smaller,

Update

Push into Priority Queue.

First time reaching destination,

Return effort.

💻 Java Code
import java.util.*;

class Tuple {
    int effort;
    int row;
    int col;

    Tuple(int effort, int row, int col) {
        this.effort = effort;
        this.row = row;
        this.col = col;
    }
}

class Solution {

    public int minimumEffortPath(int[][] heights) {

        int n = heights.length;
        int m = heights[0].length;

        int[][] effort = new int[n][m];

        for(int i=0;i<n;i++) {
            Arrays.fill(effort[i], Integer.MAX_VALUE);
        }

        PriorityQueue<Tuple> pq =
                new PriorityQueue<>(
                        (a,b)->a.effort-b.effort);

        effort[0][0] = 0;

        pq.offer(new Tuple(0,0,0));

        int[] row = {-1,1,0,0};
        int[] col = {0,0,-1,1};

        while(!pq.isEmpty()) {

            Tuple current = pq.poll();

            int diff = current.effort;
            int r = current.row;
            int c = current.col;

            if(r==n-1 && c==m-1) {
                return diff;
            }

            for(int k=0;k<4;k++) {

                int nr = r + row[k];
                int nc = c + col[k];

                if(nr>=0 && nr<n &&
                   nc>=0 && nc<m) {

                    int newEffort =
                            Math.max(
                                    diff,
                                    Math.abs(
                                            heights[r][c]
                                            - heights[nr][nc]
                                    )
                            );

                    if(newEffort < effort[nr][nc]) {

                        effort[nr][nc] = newEffort;

                        pq.offer(
                                new Tuple(
                                        newEffort,
                                        nr,
                                        nc
                                )
                        );
                    }
                }
            }
        }

        return 0;
    }
}

⏱️ Time Complexity
O(N × M × log(N × M))

Reason:

Each cell may enter the Priority Queue.

Each Priority Queue operation takes

log(N × M)

🔥 Important Points

✅ Grid problem.

✅ Edge cost is

Absolute Height Difference

✅ Path cost is

Maximum Difference

not

Sum

✅ Use Dijkstra.

✅ Priority Queue always gives the minimum effort path first.

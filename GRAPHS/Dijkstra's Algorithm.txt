🚀 Dijkstra's Algorithm
❓ What is the question asking?

We are given:

A Weighted Graph
Edge weights are:
Positive (or Zero)
A source node src

We need to find:

Shortest distance
from source
to every other vertex.
🌟 Main Idea

Since edges have different positive weights:

BFS will NOT work

We always choose the node with the:

Minimum Distance

first.

To do this efficiently, we use:

Priority Queue (Min Heap)
🌟 Why Priority Queue?

Suppose:

0 → 1 (10)
0 → 2 (2)

If we use a normal queue:

1 may be processed before 2

which is wrong.

A Priority Queue always removes:

Node with the smallest distance
🌟 What is Relaxation?

Suppose:

u --(wt)--> v

If:

dist[u] + wt < dist[v]

Then update:

dist[v] = dist[u] + wt

This is called:

Edge Relaxation
⚡ Optimal Approach
💡 Idea
Create adjacency list.
Create distance array.
Initialize all distances to:
∞ (Integer.MAX_VALUE)
Set:
dist[src] = 0
Push:
(source, 0)

into the Priority Queue.

Remove the node with the smallest distance.
Relax all its neighbors.
If a shorter distance is found:
Update distance.
Push the neighbor into the Priority Queue.
💻 Java Code
import java.util.*;

class Pair {

    int node;
    int distance;

    Pair(int node, int distance) {

        this.node = node;
        this.distance = distance;
    }
}

class Solution {

    public int[] dijkstra(int V,
                          ArrayList<ArrayList<Pair>> adj,
                          int src) {

        PriorityQueue<Pair> pq =
                new PriorityQueue<>(
                        (a,b) -> a.distance - b.distance);

        int[] dist = new int[V];

        Arrays.fill(dist, Integer.MAX_VALUE);

        dist[src] = 0;

        pq.offer(new Pair(src,0));

        while(!pq.isEmpty()) {

            Pair current = pq.poll();

            int node = current.node;

            int dis = current.distance;

            for(Pair it : adj.get(node)) {

                int adjNode = it.node;

                int wt = it.distance;

                if(dis + wt < dist[adjNode]) {

                    dist[adjNode] = dis + wt;

                    pq.offer(new Pair(adjNode,
                                      dist[adjNode]));
                }
            }
        }

        return dist;
    }
}

⏱️ Time Complexity
O((V + E) log V)

Reason:

Every Priority Queue operation takes:
O(log V)
📦 Space Complexity
Auxiliary Space
O(V)

Used for:

Distance array
Priority Queue

(The adjacency list stores the graph and is usually not counted as auxiliary space.)

🔥 Important Points

✅ Works for positive edge weights.

✅ Uses Priority Queue (Min Heap).

✅ Always processes the node with the smallest distance.

✅ Uses Edge Relaxation.

❌ Does NOT work correctly for negative edge weights.

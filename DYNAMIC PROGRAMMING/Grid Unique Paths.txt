<8. Grid Unique Paths>
❓ Problem Understanding

Imagine a robot standing at the top-left corner of a grid.

It wants to reach the bottom-right corner.

The robot can move only:

➡️ Right
⬇️ Down

Find the total number of unique paths.

Example
3 x 3 Grid

.  .  .
.  .  E

Possible Paths

R D R D

D R D R

D D R R

6

🪨 Recursive Approach
💡 Idea

To reach

(i,j)

Up
Left

Add both answers.

💻 Java Code
public class Main {

    static int solve(int i, int j) {

        if (i == 0 && j == 0)
            return 1;

        if (i < 0 || j < 0)
            return 0;

        int up = solve(i - 1, j);

        int left = solve(i, j - 1);

        return up + left;
    }

    public static void main(String[] args) {

        int m = 3;
        int n = 3;

        System.out.println(solve(m - 1, n - 1));
    }
}
▶️ Output
6

🟢 Memoization
💡 Idea

Many cells are solved repeatedly.

Store answers in

dp[i][j]

If already computed,

return it.

💻 Java Code
import java.util.Arrays;

public class Main {

    static int solve(int i, int j, int[][] dp) {

        if (i == 0 && j == 0)
            return 1;

        if (i < 0 || j < 0)
            return 0;

        if (dp[i][j] != -1)
            return dp[i][j];

        int up = solve(i - 1, j, dp);

        int left = solve(i, j - 1, dp);

        return dp[i][j] = up + left;
    }

    public static void main(String[] args) {

        int m = 3;
        int n = 3;

        int[][] dp = new int[m][n];

        for (int[] row : dp)
            Arrays.fill(row, -1);

        System.out.println(solve(m - 1, n - 1, dp));
    }
}
▶️ Output
6

💻 Java Code
public class Main {

    public static void main(String[] args) {

        int m = 3;
        int n = 3;

        int[][] dp = new int[m][n];

        dp[0][0] = 1;

        for (int i = 0; i < m; i++) {

            for (int j = 0; j < n; j++) {

                if (i == 0 && j == 0)
                    continue;

                int up = 0;
                int left = 0;

                if (i > 0)
                    up = dp[i - 1][j];

                if (j > 0)
                    left = dp[i][j - 1];

                dp[i][j] = up + left;
            }
        }

        System.out.println(dp[m - 1][n - 1]);
    }
}
▶️ Output
6
⏱ Time Complexity
O(m × n)
📦 Space Complexity
O(m × n)
⚡ Space Optimization
💡 Idea

Observe:

depends only on

Current Row

and

Previous Row

So,

instead of storing the entire grid,

store only:

prev[]

curr[]
💻 Java Code
public class Main {

    public static void main(String[] args) {

        int m = 3;
        int n = 3;

        int[] prev = new int[n];

        for (int i = 0; i < m; i++) {

            int[] curr = new int[n];

            for (int j = 0; j < n; j++) {

                if (i == 0 && j == 0) {

                    curr[j] = 1;

                } else {

                    int up = 0;
                    int left = 0;

                    if (i > 0)
                        up = prev[j];

                    if (j > 0)
                        left = curr[j - 1];

                    curr[j] = up + left;
                }
            }

            prev = curr;
        }

        System.out.println(prev[n - 1]);
    }
}
▶️ Output
6
⏱ Time Complexity
O(m × n)
📦 Space Complexity
O(n)

Every DP state is identified by two coordinates, not just one index.

🎯 Didu's Golden Trick 🌸

Whenever you see a Grid DP problem, always ask these 3 questions:

What is my DP state?
Usually dp[row][col].
From where can I come to this cell?
In this problem: Up and Left.
What should I store?
Here: Number of ways to reach this cell.

If you answer these three questions first, writing the recurrence becomes almost automatic.

💖 Super Memory Formula

1D DP
↓

State = Index

Grid DP
↓

State = (Row, Column)

↓

Answer at Current Cell

=

Answer from Possible Previous Cells

This single idea is the foundation for Unique Paths, Unique Paths II, Minimum Path Sum, Triangle, Maximum Falling Path Sum, and many other grid DP problems. Master this one, and the rest of Grid DP will feel much more natural. 🌟🚀

<2. Climbing Stairs>
❓ What is the question asking?

We are standing at the 0th stair and want to reach the Nth stair.

At every move, we have only 2 choices:

Climb 1 stair
Climb 2 stairs

We need to find how many different ways we can reach the Nth stair.

📝 Example
Input
n = 3

Possible ways:

1 → 1 → 1

2 → 1

1 → 2

Output

3
🧠 Observation

To reach stair n, we can come from:

Stair n-1 (take 1 step)
Stair n-2 (take 2 steps)

So,

Ways(n) = Ways(n-1) + Ways(n-2)

This is exactly the Fibonacci Pattern.

The only difference is the base cases.

Ways(0) = 1
Ways(1) = 1
🪨 Recursive Approach
💡 Idea

For every stair:

Jump 1 step.
Jump 2 steps.
Add both answers.
📦 Java Code
public class Main {

    static int climbStairs(int n) {

        if (n == 0 || n == 1)
            return 1;

        return climbStairs(n - 1) + climbStairs(n - 2);
    }

    public static void main(String[] args) {

        int n = 3;

        System.out.println(climbStairs(n));

    }
}
▶️ Output
3
⏱ Time Complexity
O(2^N)

Many subproblems are solved repeatedly.

📦 Space Complexity
O(N)

Recursion stack.

🟢 Memoization (Top-Down DP)
💡 Idea

Use recursion, but before solving a stair, check whether its answer is already stored in the DP array.

If yes → return it.
Otherwise → calculate it, store it, and return it.

This avoids recalculating the same subproblems.

📦 Java Code
import java.util.Arrays;

public class Main {

    static int climbStairs(int n, int[] dp) {

        if (n == 0 || n == 1)
            return 1;

        if (dp[n] != -1)
            return dp[n];

        dp[n] = climbStairs(n - 1, dp) + climbStairs(n - 2, dp);

        return dp[n];
    }

    public static void main(String[] args) {

        int n = 3;

        int[] dp = new int[n + 1];

        Arrays.fill(dp, -1);

        System.out.println(climbStairs(n, dp));

    }
}
▶️ Output
3
⏱ Time Complexity
O(N)

Each stair is solved only once.

📦 Space Complexity
O(N)

O(N)

🟡 Tabulation (Bottom-Up DP)
💡 Idea

Instead of recursion, start from the base cases and build the answer iteratively.

Steps:

Create a DP array.
Initialize:
dp[0] = 1
dp[1] = 1
For every stair from 2 to n:
dp[i] = dp[i-1] + dp[i-2]
Return dp[n].
📦 Java Code
public class Main {

    static int climbStairs(int n) {

        if (n == 0 || n == 1)
            return 1;

        int[] dp = new int[n + 1];

        dp[0] = 1;
        dp[1] = 1;

        for (int i = 2; i <= n; i++) {

            dp[i] = dp[i - 1] + dp[i - 2];

        }

        return dp[n];
    }

    public static void main(String[] args) {

        int n = 3;

        System.out.println(climbStairs(n));

    }
}
▶️ Output
3
⏱ Time Complexity
O(N)

Single loop.

📦 Space Complexity
O(N)

DP array.

⚡ Space Optimization
💡 Idea

Observe the recurrence relation:

dp[i] = dp[i-1] + dp[i-2]

To calculate the current stair, we only need the previous two answers.

Instead of storing the entire DP array, keep only:

prev → Ways to reach (i-1)th stair.
prev2 → Ways to reach (i-2)th stair.

For every iteration:

Calculate the current answer.
Update prev2.
Update prev.

This reduces space complexity to O(1).

📦 Java Code
public class Main {

    static int climbStairs(int n) {

        if (n == 0 || n == 1)
            return 1;

        int prev2 = 1;
        int prev = 1;

        for (int i = 2; i <= n; i++) {

            int curr = prev + prev2;

            prev2 = prev;
            prev = curr;
        }

        return prev;
    }

    public static void main(String[] args) {

        int n = 3;

        System.out.println(climbStairs(n));

    }
}
▶️ Output
3
⏱ Time Complexity
O(N)
📦 Space Complexity
O(1)

Only three variables are used.

This is the same recurrence as Fibonacci.

The only difference is the base cases:

Fibonacci	Climbing Stairs
F(0) = 0	Ways(0) = 1

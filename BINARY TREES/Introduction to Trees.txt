🌳 Introduction to Trees 💖
🌟 What is a Tree?

A Tree is a non-linear data structure used to store data in a hierarchical way.

It looks like an upside-down tree 🌳

Example:

        1
       / \
      2   3
     / \
    4   5
🧠 Real Life Examples

Trees are everywhere 👀

File folders in laptop 📁
Family tree 👨‍👩‍👧
Organization hierarchy 🏢
HTML DOM 🌐
🌳 Important Terminologies
🔹 Root Node

Topmost node.

Root = 1
🔹 Parent Node

A node having children.

1 is parent of 2 and 3
🔹 Child Node

Node connected below parent.

2 and 3 are children of 1
🔹 Leaf Node

Node with NO children.

4, 5, 3
🔹 Siblings

Nodes having same parent.

2 and 3
🔹 Subtree

Smaller tree inside tree.

Example:

    2
   / \
  4   5

is subtree of main tree.

🌟 Binary Tree

A tree where each node has at most:

2 children
Left child
Right child
🌳 Structure of Binary Tree
class Node {

    int data;

    Node left;
    Node right;

    Node(int data) {
        this.data = data;
        left = right = null;
    }
}
🌟 Types of Binary Trees
1️⃣ Full Binary Tree

Every node has:

0 children OR
2 children
2️⃣ Complete Binary Tree

All levels filled except maybe last level.

Last level filled from left to right.

3️⃣ Perfect Binary Tree

All leaf nodes at same level.

Every level completely filled.

4️⃣ Balanced Binary Tree

Height difference between left and right subtree is small.

Searching becomes fast ⚡

5️⃣ Degenerate Tree

Looks like linked list 😭

1
 \
  2
   \
    3
🌟 Traversals in Trees

Traversal means visiting all nodes.

🔹 Preorder
Root → Left → Right
🔹 Inorder
Left → Root → Right
🔹 Postorder
Left → Right → Root

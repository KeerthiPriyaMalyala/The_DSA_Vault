🌳 Binary Tree Zigzag Level Order Traversal 💖
❓ What is the question asking?

Traverse the binary tree:

level by level

BUT direction changes at every level 😎

🌟 Zigzag Order
Level 1
Left → Right
Level 2
Right → Left
Level 3
Left → Right

and so on...

🌳 Example Tree
        1
       / \
      2   3
     / \   \
    4   5   6
✅ Zigzag Output
[
 [1],
 [3,2],
 [4,5,6]
]
🌟 Main Idea 💡

We use:

Queue (Level Order Traversal)

AND a boolean variable:

leftToRight

to decide direction.

🌟 Algorithm
💡 Steps

1️⃣ Perform normal level order traversal
2️⃣ Store current level nodes
3️⃣ If direction reversed:

insert at beginning
OR reverse list
4️⃣ Toggle direction
⚡ Optimal Approach (Queue + Deque)
💡 Why Deque?

Because:

we can add at front OR back

efficiently 😎

💻 Java Code
import java.util.*;

class Solution {

    public List<List<Integer>> zigzagLevelOrder(TreeNode root) {

        List<List<Integer>> result = new ArrayList<>();

        if (root == null)
            return result;

        Queue<TreeNode> queue = new LinkedList<>();

        queue.offer(root);

        boolean leftToRight = true;

        while (!queue.isEmpty()) {

            int size = queue.size();

            Deque<Integer> level = new LinkedList<>();

            for (int i = 0; i < size; i++) {

                TreeNode current = queue.poll();

                if (leftToRight) {

                    level.addLast(current.val);
                }
                else {

                    level.addFirst(current.val);
                }

                if (current.left != null) {
                    queue.offer(current.left);
                }

                if (current.right != null) {
                    queue.offer(current.right);
                }
            }

            result.add(new ArrayList<>(level));

            leftToRight = !leftToRight;
        }

        return result;
    }
}

⏱️ Time Complexity
Complexity	Value
Time	O(n)

Every node visited once.

📦 Space Complexity
Complexity	Value
Space	O(n)

Queue + result list.

🌟 Important Observation

Normal level order:

Always Left → Right

Zigzag:

Direction alternates
🌟 Alternate Simpler Approach

Instead of deque:

Store level normally
Reverse list when needed

But reversing costs extra time.

Deque approach is cleaner 😎

312. Sorted GCD Pair Queries (Hard)
❓ What is the question asking?

You are given:

an array nums
an array queries
Step 1

Generate GCD of every possible pair.

Example:

nums = [2,3,4]

Pairs:
(2,3) -> 1
(2,4) -> 2
(3,4) -> 1

gcdPairs = [1,2,1]
Step 2

Sort them.

[1,1,2]
Step 3

For every query,

query = 0
answer = gcdPairs[0] = 1

query = 2
answer = gcdPairs[2] = 2
🪨 Brute Force Approach
💡 Idea

Generate every pair.

for i
   for j
      gcd(nums[i],nums[j])

Store all GCDs.

Sort them.

Answer each query.

💻 Java Code
import java.util.*;

public class Main {

    static int gcd(int a, int b) {
        while (b != 0) {
            int t = a % b;
            a = b;
            b = t;
        }
        return a;
    }

    public static int[] gcdValues(int[] nums, long[] queries) {

        ArrayList<Integer> list = new ArrayList<>();

        for (int i = 0; i < nums.length; i++) {
            for (int j = i + 1; j < nums.length; j++) {
                list.add(gcd(nums[i], nums[j]));
            }
        }

        Collections.sort(list);

        int[] ans = new int[queries.length];

        for (int i = 0; i < queries.length; i++) {
            ans[i] = list.get((int) queries[i]);
        }

        return ans;
    }

    public static void main(String[] args) {

        int nums[] = {2,3,4};
        long queries[] = {0,2};

        System.out.println(Arrays.toString(gcdValues(nums, queries)));
    }
}

[1,2]

The brute force cannot work because

💡 Main Observation

Instead of generating every pair,

we count

How many pairs have GCD exactly = x

for every possible value of x.

💻 Java Code (Optimal)
import java.util.*;

class Solution {

    public int[] gcdValues(int[] nums, long[] queries) {

        int max = 0;

        for (int x : nums)
            max = Math.max(max, x);

        int[] freq = new int[max + 1];

        for (int x : nums)
            freq[x]++;

        long[] exact = new long[max + 1];

        // exact[g] = number of pairs having gcd exactly g
        for (int g = max; g >= 1; g--) {

            long count = 0;

            for (int multiple = g; multiple <= max; multiple += g)
                count += freq[multiple];

            exact[g] = count * (count - 1) / 2;

            for (int multiple = g * 2; multiple <= max; multiple += g)
                exact[g] -= exact[multiple];
        }

        long[] prefix = new long[max + 1];

        for (int g = 1; g <= max; g++)
            prefix[g] = prefix[g - 1] + exact[g];

        int[] ans = new int[queries.length];

        for (int i = 0; i < queries.length; i++) {

            long q = queries[i] + 1; // convert 0-index to 1-index

            int left = 1, right = max;

            while (left < right) {

                int mid = (left + right) / 2;

                if (prefix[mid] >= q)
                    right = mid;
                else
                    left = mid + 1;
            }

            ans[i] = left;
        }

        return ans;
    }
}

Counting exact GCD pairs using multiples:

Binary search for each query:

O(q log MAX)

Overall:

O(n + MAX log MAX + q log MAX)
